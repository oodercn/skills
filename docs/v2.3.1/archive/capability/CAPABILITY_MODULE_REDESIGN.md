## 一、设计理念

### 1.1 SceneSkills 核心概念

**场景能力定义**：
- **开箱即用** - 用户无需配置即可使用
- **具备用户交互UI** - 有完整的用户界面
- **可在导航菜单显示** - 作为独立功能入口

**能力分层**：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           能力分层架构                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  第一层：场景能力 (Scene Capabilities)                               │   │
│  │  - 定义：开箱即用 + 用户交互UI + 导航菜单入口                          │   │
│  │  - 示例：日志汇报、LLM对话、知识问答、项目管理                          │   │
│  │  - 特点：面向终端用户，提供完整业务解决方案                             │   │
│  │  - 数据来源：skill-index.yaml 中的 scenes 列表                        │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  第二层：技能能力 (Skill Capabilities)                               │   │
│  │  - 定义：底层能力组件，提供原子化功能                                  │   │
│  │  - 示例：日志提醒、日志提交、邮件通知、文件存储                          │   │
│  │  - 特点：面向开发者，可被场景能力组合使用                               │   │
│  │  - 数据来源：skill-index.yaml 中的 skills 列表                        │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                    │                                        │
│                                    ▼                                        │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  第三层：基础服务 (Infrastructure Services)                          │   │
│  │  - 定义：系统级基础设施服务                                           │   │
│  │  - 示例：认证服务、消息队列、存储服务、监控服务                          │   │
│  │  - 特点：面向运维，提供系统级能力                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 设计原则

1. **场景优先** - 以场景能力为主要入口，技能能力作为支撑
2. **用户导向** - 场景能力面向终端用户，技能能力面向开发者
3. **开箱即用** - 场景能力安装后即可使用，无需额外配置
4. **能力组合** - 一个场景能力可组合多个技能能力
