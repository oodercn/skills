# 需求驱动的技能分类框架

## 一、需求场景分析

### 场景1：我要找一个招聘APP

```
用户视角：最终用户
目的：发现和使用场景
搜索路径：
  ├── 按业务领域：HR → 招聘
  ├── 按场景类型：场景技能
  └── 按关键词：招聘、人才、面试

期望结果：
  - skill-recruitment-assistant (招聘助手场景)
  - skill-interview-scheduler (面试安排场景)
```

### 场景2：我需要将招聘添加一个邮件自动发送的功能

```
用户视角：开发者/管理员
目的：扩展场景能力
搜索路径：
  ├── 按功能分类：COMMUNICATION → 邮件
  ├── 按能力类型：独立技能（可被场景调用）
  └── 按关键词：邮件、通知、发送

期望结果：
  - skill-email (邮件服务) - 独立技能，可被场景调用
  - skill-notify (通知服务) - 独立技能，支持多渠道
```

### 场景3：我要为招聘的模块和审批打通

```
用户视角：开发者/管理员
目的：场景协同 + 数据集成
需求分解：
  ├── 3.1 建立OA场景和招聘场景的协同关系
  │   └── 需要找到：场景协同配置能力
  └── 3.2 添加一个数据通向的能力组件
      └── 需要找到：数据集成/数据通道组件

搜索路径：
  ├── 按功能分类：WORKFLOW → 审批
  ├── 按能力类型：场景协同
  └── 按关键词：审批、流程、协同

期望结果：
  - skill-approval-workflow (审批流程场景)
  - skill-data-connector (数据连接器) - 独立技能
  - skill-scene-collaboration (场景协同服务)
```

## 二、需求驱动的分类设计

### 2.1 从需求提取分类维度

| 需求 | 提取的维度 | 说明 |
|------|------------|------|
| 场景1：找招聘APP | **业务领域** + **场景标识** | 用户按业务领域发现场景 |
| 场景2：添加邮件功能 | **功能分类** + **独立能力标识** | 开发者按功能发现可复用能力 |
| 场景3：打通审批 | **场景协同** + **数据通道** | 场景间关系和数据集成 |

## 三、场景映射

### 3.1 场景1：找招聘APP

```yaml
# 招聘助手场景
- skillId: skill-recruitment-assistant
  name: 招聘助手
  type: SCENE
  category: WORKFLOW
  domain: hr
  subCategory: recruitment
  tags:
    - 招聘
    - 人才
    - 面试
  description: 智能招聘助手场景，支持简历筛选、面试安排、候选人管理
  sceneType: AUTO
  visibility: PUBLIC
```

**用户搜索路径**:
1. 按业务领域：`domain: hr` → 找到所有HR相关技能
2. 按场景类型：`type: SCENE` → 筛选出场景技能
3. 按关键词：`招聘` → 找到招聘助手

## 四、能力类型详解

### 4.1 SCENE (场景技能)

| 属性 | 说明 |
|------|------|
| **面向用户** | 最终用户 |
| **使用方式** | 需要激活 |
| **特点** | 有业务语义、有参与者、有可见性 |
| **示例** | 招聘助手、会议纪要、文档问答 |

### 4.2 PROVIDER (提供者)

| 属性 | 说明 |
|------|------|
| **面向用户** | 开发者/场景 |
| **使用方式** | 独立运行，可被调用 |
| **特点** | 提供基础能力，可复用 |
| **示例** | 邮件服务、存储服务、LLM服务 |

### 4.3 INTERNAL (内部能力)

| 属性 | 说明 |
|------|------|
| **面向用户** | 其他能力 |
| **使用方式** | 被调用，不独立运行 |
| **特点** | 工具类，无独立业务语义 |
| **示例** | 上下文构建器、文档处理器 |

### 4.4 CONNECTOR (连接器)

| 属性 | 说明 |
|------|------|
| **面向用户** | 开发者/管理员 |
| **使用方式** | 配置场景间关系 |
| **特点** | 场景间数据通道、协同配置 |
| **示例** | 数据连接器、场景协同服务 |

## 五、搜索场景支持

### 5.1 按用户角色搜索

| 角色 | 搜索路径 | 示例 |
|------|----------|------|
| **最终用户** | domain → type:SCENE | HR → 招聘助手 |
| **开发者** | category → type:PROVIDER | COMMUNICATION → 邮件服务 |
| **管理员** | type:CONNECTOR | 场景协同服务 |

### 5.2 按需求场景搜索

| 需求 | 搜索条件 | 结果 |
|------|----------|------|
| 找场景 | `type:SCENE` + `domain:hr` | 招聘助手 |
| 找能力 | `type:PROVIDER` + `category:COMMUNICATION` | 邮件服务 |
| 找连接器 | `type:CONNECTOR` | 数据连接器 |
