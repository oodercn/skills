## 三、概念层次重构

### 3.1 旧概念层次

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        旧概念层次结构                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   第一层：场景                                                               │
│   ─────────────                                                              │
│   SceneDefinition (场景定义)                                                │
│   ├── capabilities: List<Capability>    ← 场景包含能力                      │
│   ├── roles: List<Role>                 ← 角色定义                          │
│   ├── workflow: WorkflowDefinition      ← 工作流定义                        │
│   └── triggers: List<Trigger>           ← 触发器                            │
│                                                                             │
│   第二层：能力                                                               │
│   ─────────────                                                              │
│   Capability (能力)                                                          │
│   ├── capabilityId: String                                                   │
│   ├── type: CapabilityType                                                   │
│   └── ...                                                                    │
│                                                                             │
│   第三层：场景组                                                             │
│   ─────────────                                                              │
│   SceneGroup (场景组)                                                        │
│   ├── participants: List<Participant>                                        │
│   ├── capabilityBindings: List<CapabilityBinding>                           │
│   └── status: SceneGroupStatus                                               │
│                                                                             │
│   问题：                                                                     │
│   ├── 场景和能力是两个独立概念                                              │

```

## 四、术语重定义汇总表

### 4.1 核心术语重定义

| 序号 | 旧术语 | 新术语 | 定义变化 | 来源 |
|------|--------|--------|----------|------|
| 1 | SceneDefinition | SceneCapability | 场景定义变为场景能力，场景本身就是一种能力 | 能力驱动理论 |
| 2 | primaryScene | mainFirst | 主场景变为自驱入口，强调主动驱动 | 用户故事：场景如何启动 |
| 3 | collaborativeScenes | collaborativeCapabilities | 协作场景变为协作能力，统一能力概念 | 能力驱动理论 |
| 4 | WorkflowDefinition | capabilityChains | 工作流变为能力调用链，用能力链替代工作流 | 用户故事：能力调用顺序 |
| 5 | Trigger | DriverCapability | 触发器变为驱动能力，统一驱动概念 | 用户故事：时间/事件触发 |
| 6 | 涌现能力 | 场景能力涌现 | 涌现能力是场景能力协调子能力产生的 | 用户故事：LLM协调分析 |
| 7 | 场景创建 | 能力自启 | 创建场景变为能力自启动 | 用户故事：零配置安装 |
| 8 | 场景激活 | 能力自驱 | 激活场景变为能力自驱运行 | 用户故事：场景持续运行 |

### 4.2 新增术语定义

| 序号 | 新术语 | 英文标识 | 定义 | 来源 |
|------|--------|----------|------|------|
| 1 | 场景能力 | SceneCapability | 自驱型SuperAgent能力，包含子能力和驱动能力 | 能力驱动理论 |
| 2 | 自驱入口 | mainFirst | 场景能力的启动入口，包含自检、自启、自驱 | 用户故事：场景如何启动 |
| 3 | 驱动能力 | DRIVER_CAPABILITY | 提供驱动源头的特殊能力类型 | 用户故事：时间/事件/意图触发 |
| 4 | 意图接收 | intent-receiver | 接收用户意图并触发场景启动的驱动能力 | 用户故事：用户需求驱动 |
| 5 | 时间驱动 | scheduler | 监听时间事件并触发能力调用的驱动能力 | 用户故事：定时提醒 |
| 6 | 事件监听 | event-listener | 监听业务事件并触发能力调用的驱动能力 | 用户故事：条件触发 |
| 7 | 能力调用 | capability-invoker | 管理能力调用链的驱动能力 | 用户故事：能力调用顺序 |
| 8 | 协作协调 | collaboration-coordinator | 协调协作场景的驱动能力 | 用户故事：多场景协作 |
| 9 | 能力调用链 | capabilityChains | 能力的有序调用序列，支持条件和分支 | 用户故事：工作流编排 |
| 10 | 原子能力 | ATOMIC_CAPABILITY | 单一功能、不可分解的能力 | 能力分类 |
| 11 | 组合能力 | COMPOSITE_CAPABILITY | 组合多个原子能力的能力 | 能力分类 |
