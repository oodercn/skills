## 典型用例详解

### 用例一：日志汇报场景（ABS）

**完整故事**：详见 [STORY_DAILY_REPORT.md](./STORY_DAILY_REPORT.md)

**核心流程**：
```
17:00 系统自动发送提醒 → 员工提交日志 → 20:00 系统自动汇总 → AI分析生成报告
```

**能力链**：
1. report-remind（日志提醒）
2. report-submit（日志提交）
3. report-aggregate（日志汇总）
4. report-analyze（日志分析）

---

### 用例二：系统清理场景（ASS）

**用户故事**：
> 我是系统管理员，我希望系统每天凌晨2:00自动清理临时文件和日志，保持系统健康。

**分类**: ASS（自驱系统场景）

**驱动方式**: scheduler

**能力链**：
1. temp-file-clean（临时文件清理）
2. log-rotate（日志轮转）
3. cache-clear（缓存清理）

---

### 用例三：审批流程场景（TBS）

**用户故事**：
> 我是部门主管，我希望员工提交的申请需要我审批后才能生效。

**分类**: TBS（触发业务场景）

**触发方式**: 人工/API

**能力链**：
1. application-submit（申请提交）
2. approval-notify（审批通知）
3. approval-process（审批处理）
4. result-notify（结果通知）
