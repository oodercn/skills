## 一、需求分析

### 1.1 用户角色与场景

| 角色 | 核心需求 | 主要场景 | 能力需求 |
|------|---------|---------|---------|
| **系统管理员** | 系统配置和维护 | 安装技能、配置驱动、监控系统 | 系统管理、监控告警、安全审计 |
| **业务用户** | 使用场景完成业务目标 | 激活场景、配置参数、查看结果 | 场景发现、场景激活、结果查看 |
| **开发者** | 开发技能包 | 编写技能、定义能力、发布技能 | 开发工具、测试框架、发布流程 |

### 1.2 核心业务场景

根据用户闭环故事，识别出以下核心业务场景：

#### 场景1: 能力管理闭环

```
发现能力 → 激活能力 → 配置能力 → 使用能力 → 查看统计 → 停用能力
```

**能力需求**:
- 能力发现与注册
- 能力激活与绑定
- 能力配置管理
- 能力使用统计

#### 场景2: 场景管理闭环

```
发现场景 → 激活场景 → 配置场景 → 运行场景 → 查看结果 → 停用场景
```

**能力需求**:
- 场景发现与安装
- 场景激活与配置
- 场景运行与监控
- 场景结果展示

#### 场景3: 配置管理闭环

```
配置LLM → 测试连接 → 配置知识库 → 上传文档 → 绑定场景组
```

**能力需求**:
- LLM配置与测试
- 知识库管理
- 文档处理与索引
- 场景组配置

### 2.2 分类定义

#### 核心层 (Core Layer) - MVP内置

| 分类 | 能力 | 职责 | 状态 |
|------|------|------|------|
| **能力管理** | Capability Management | 能力注册、发现、激活、绑定 | MVP内置 |
| **场景管理** | Scene Management | 场景安装、激活、运行、监控 | MVP内置 |
| **LLM核心** | LLM Core | LLM调用、上下文管理、Function Calling | MVP内置 |
| **知识库** | Knowledge Base | 文档管理、向量索引、RAG检索 | MVP内置 |
| **审计服务** | Audit Service | 操作审计、日志记录、合规检查 | MVP内置 |
| **安全服务** | Security Service | 权限控制、数据加密、安全策略 | MVP内置 |
| **Agent管理** | Agent Management | Agent注册、心跳、调度 | MVP内置 |
| **用户认证** | User Authentication | 用户登录、会话管理、角色权限 | MVP内置 |

#### 驱动层 (Driver Layer) - 第三方集成

| 分类 | 驱动 | 提供者 | 依赖 |
|------|------|--------|------|
| **LLM驱动** | DeepSeek, OpenAI, Qianwen, Volcengine, Ollama, Baidu | 各模型厂商 | LLM核心 |
| **存储驱动** | VFS-Base, VFS-Local, VFS-S3, VFS-OSS, VFS-Minio, VFS-Database | 各云厂商 | - |
| **组织驱动** | Org-Base, Org-Feishu, Org-Wecom, Org-Dingding, Org-LDAP | 各平台 | 用户认证 |
| **通讯驱动** | IM-Dingding, IM-Feishu, IM-Wecom, Email, MQTT | 各平台 | - |
| **支付驱动** | Payment-Alipay, Payment-Wechat, Payment-Unionpay | 各支付渠道 | - |
| **媒体驱动** | Media-WeChat, Media-Weibo, Media-Zhihu, Media-Toutiao, Media-Xiaohongshu | 各媒体平台 | - |

#### 服务层 (Service Layer) - 可插拔服务

| 分类 | 服务 | 能力 | 依赖 |
|------|------|------|------|
| **通信服务** | Email, IM, Notify, Msg, Group, MQTT | 消息发送、通知推送、群组管理 | 通讯驱动 |
| **监控服务** | Health, Monitor, Network, Remote-Terminal, Cmd-Service, Res-Service | 健康检查、性能监控、远程管理 | - |
| **调度服务** | Scheduler-Quartz, Task | 定时任务、任务调度 | - |
| **搜索服务** | Search | 全文搜索、语义搜索 | 知识库 |
| **认证服务** | User-Auth | 扩展认证方式 | 用户认证 |
| **基础设施** | OpenWrt, K8s | 基础设施管理 | - |
| **工具服务** | Common, Protocol, Management | 通用工具、协议处理、技能管理 | - |

#### 场景层 (Scene Layer) - 业务场景

| 场景类型 | 场景 | 能力需求 | 用户角色 |
|---------|------|---------|---------|
| **AUTO + public** | LLM聊天助手、知识问答、日报生成 | LLM核心、知识库、文档处理 | 业务用户 |
| **AUTO + internal** | 健康监控、更新检查、系统调度 | 监控服务、调度服务 | 系统管理员 |
| **TRIGGER + public** | 文档助手、会议纪要、入职培训、项目知识、知识分享、协作场景 | LLM核心、知识库、文档处理、通知服务 | 业务用户 |

## 六、能力库治理规范

### 6.1 能力注册规范

每个能力必须包含：

1. **能力定义** (skill.yaml)
   - 能力ID、名称、版本
   - 能力描述、分类、标签
   - 能力接口、参数、返回值

2. **能力说明书** (README.md)
   - 能力概述
   - 使用场景
   - 参数说明
   - 示例代码
   - 最佳实践

3. **能力测试** (test/)
   - 单元测试
   - 集成测试
   - 性能测试

### 6.2 能力依赖规范

1. **显式声明**: 所有依赖必须在skill.yaml中显式声明
2. **版本约束**: 使用语义化版本约束 (>=1.0.0, <2.0.0)
3. **依赖检查**: 安装时自动检查依赖是否满足
4. **循环依赖**: 禁止循环依赖

### 6.3 能力生命周期管理
