## 二、MVP 工程职责

### 2.1 核心职责

1. **运行时容器**：提供 Spring Boot 运行环境
2. **模块管理**：模块的安装、卸载、激活、停用
3. **配置管理**：三级配置继承体系
4. **安全认证**：用户认证和授权
5. **API 网关**：统一 API 入口

### 2.2 内置功能

| 功能 | 说明 | 来源 |
|------|------|------|
| nexus.css | 前端框架 | MVP 内置 |
| nexus.js | JS 核心 | MVP 内置 |
| menu.js | 菜单系统 | MVP 内置 |
| auth 模块 | 认证授权 | skill-common |
| config 模块 | 配置管理 | skill-common |
| org 模块 | 组织管理 | skill-common |

## 三、Skills 职责

### 3.1 核心职责

1. **模块仓库**：存储可安装的功能模块
2. **能力提供**：提供各种能力服务
3. **驱动适配**：适配第三方系统和服务
4. **场景定义**：定义业务场景

### 3.2 Skill 类型

| 类型 | 目录 | 说明 | 示例 |
|------|------|------|------|
| system | _system/ | 系统核心 | skill-common, skill-capability |
| capability | capabilities/ | 能力服务 | skill-knowledge, skill-llm-conversation |
| driver | _drivers/ | 驱动适配 | skill-llm-deepseek, skill-vfs-local |
| scene | skill-scene/ | 场景引擎 | capability-discovery, scene-management |

## 四、模块安装流程

### 4.1 安装流程图

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  发现模块   │────▶│  检查依赖   │────▶│  复制资源   │────▶│  注册模块   │
└─────────────┘     └─────────────┘     └─────────────┘     └─────────────┘
      │                   │                   │                   │
      ▼                   ▼                   ▼                   ▼
 扫描 skills/        检查依赖是否        复制前端资源        更新模块注册表
 目录中的模块        满足条件           和后端代码          添加菜单项
```

### 4.2 安装 API

```java
// 发现可用模块
GET /api/v1/module/discover?source=local|git|market

// 安装模块
POST /api/v1/module/install
{
  "moduleId": "capability-discovery",
  "source": "skills/skill-scene",
  "config": {
    "role": "installer",
    "participants": []
  }
}

// 卸载模块
DELETE /api/v1/module/{moduleId}

// 激活/停用模块
PUT /api/v1/module/{moduleId}/status
{
  "status": "active|inactive"
}
```

### 4.3 安装步骤

1. **扫描发现**
   - 扫描 skills 目录
   - 解析 skill.json 和 module.json
   - 返回可用模块列表

2. **依赖检查**
   - 检查模块依赖是否满足
   - 必需依赖缺失则安装失败
   - 可选依赖缺失则警告

3. **资源复制**
   - 复制前端资源到 static/modules/{moduleId}/
   - 复制后端代码（如果需要）
   - 合并配置文件

4. **注册模块**
   - 更新模块注册表
   - 添加菜单项
   - 初始化模块配置

## 五、Skills 改造建议

### 5.1 当前问题

1. **模块边界不清晰**：skill-scene 包含太多功能
2. **资源分散**：页面、JS、CSS 分散在不同目录
3. **依赖关系复杂**：模块间依赖不明确
4. **安装机制缺失**：没有统一的安装流程

### 5.2 改造方案

#### 方案一：模块化重构

```
skill-scene/
├── skill.json
├── modules/
│   ├── capability-discovery/
│   │   ├── module.json
│   │   ├── frontend/
│   │   │   ├── capability-discovery.html
│   │   │   ├── capability-discovery.js
│   │   │   └── capability-discovery.css
│   │   └── backend/
│   │       └── CapabilityDiscoveryController.java
│   ├── scene-management/
│   │   └── ...
│   └── llm-config/
│       └── ...
└── src/main/java/
    └── net/ooder/skill/scene/
        └── shared/              # 共享代码
            ├── service/
            └── dto/
```

#### 方案二：按能力拆分

```
skills/
├── skill-scene-core/           # 场景核心
│   └── modules/
│       ├── scene-management/
│       └── template-management/
│
├── skill-capability-discovery/ # 能力发现（独立）
│   └── modules/
│       └── capability-discovery/
│
├── skill-llm-config/           # LLM配置（独立）
│   └── modules/
│       └── llm-config/
│
└── skill-knowledge/            # 知识库
    └── modules/
        └── knowledge-base/
```

### 5.3 改造步骤

1. **Phase 1：定义规范**
   - 制定 module.json 规范
   - 制定 skill.json 规范
   - 定义模块目录结构

2. **Phase 2：重构 skill-scene**
   - 按功能拆分模块
   - 添加模块元数据
   - 整理依赖关系

3. **Phase 3：实现安装机制**
   - 实现模块发现 API
   - 实现模块安装 API
   - 实现模块卸载 API

4. **Phase 4：验证测试**
   - 测试模块安装流程
   - 测试依赖检查
   - 测试配置继承
