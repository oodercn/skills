# 日志汇报场景测试指南

## 一、测试环境准备

### 1.1 启动应用

```bash
cd e:\github\ooder-skills\skills\skill-scene
mvn spring-boot:run -DskipTests
```

等待启动成功，看到以下日志表示启动完成：
```
Started SceneSkillApplication in X.XXX seconds
```

### 1.2 访问地址

- 主页面: http://localhost:8084/console/pages/scene-management.html
- 模板管理: http://localhost:8084/console/pages/template-management.html
- 场景组管理: http://localhost:8084/console/pages/scene-group-management.html

### 测试用例9：调用日志汇报Skill API

**目标**: 测试Skill API接口

**步骤**:

| 步骤 | 工具 | 操作 | 预期结果 |
|------|------|------|---------|
| 1 | PowerShell | `Invoke-RestMethod -Uri "http://localhost:8084/api/skills/daily-report/capabilities" -Method GET` | 返回5个能力定义 |
| 2 | PowerShell | 发送提醒请求 (见下方) | 返回成功，包含reminderId |
| 3 | PowerShell | 提交日志请求 (见下方) | 返回成功，包含reportId |
| 4 | PowerShell | 汇总日志请求 (见下方) | 返回汇总结果 |
| 5 | PowerShell | 分析日志请求 (见下方) | 返回分析结果 |

**API请求示例**:

```powershell
# 发送提醒
Invoke-RestMethod -Uri "http://localhost:8084/api/skills/daily-report/remind" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"sceneGroupId":"sg-test","targetUsers":["user-001","user-002"],"message":"请提交日志"}'

# 提交日志
Invoke-RestMethod -Uri "http://localhost:8084/api/skills/daily-report/submit" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"sceneGroupId":"sg-test","userId":"user-001","content":"今日完成开发任务"}'

# 汇总日志
Invoke-RestMethod -Uri "http://localhost:8084/api/skills/daily-report/aggregate" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"sceneGroupId":"sg-test"}'

# 分析日志
Invoke-RestMethod -Uri "http://localhost:8084/api/skills/daily-report/analyze" `
  -Method POST `
  -ContentType "application/json" `
  -Body '{"sceneGroupId":"sg-test","analyzeType":"summary"}'
```

## 六、常见问题排查

### Q1: 页面404错误
**原因**: 静态资源未加载
**解决**: 重启应用 `mvn spring-boot:run`

### Q2: API返回空数据
**原因**: 测试数据未初始化
**解决**: 检查 `DailyReportTestDataInitializer` 是否执行

### Q3: 按钮点击无响应
**原因**: JS文件未加载或报错
**解决**: 打开浏览器开发者工具查看Console错误
