Apex：新一代 AI Agent 基础设施

当传统 IT 基建遇见大模型，新旧技术的火花在此涌现

一、Apex 释义

Apex = 顶点 + 融合点

我们正站在一个特殊的时代节点：传统 IT 技术经过数十年沉淀，已足够成熟稳定；大模型技术方兴未艾，正在重塑一切。

Apex 正是这两股力量的交汇处：

核心理念

颠覆从来不是推翻，而是在基础设施足够完善之后，新旧技术的火花自然涌现。

二、一句话定位

Apex 是一个 MIT 开源的场景驱动 AI Agent 基础设施，致力于让 AI 能力像乐高一样自由组合。

三、核心架构

三大核心概念

概念 定义 价值
场景 业务上下文容器 = 参与者 + 能力 + 知识库 + LLM 让 AI 能力有业务语境
技能 能力封装单元，声明式定义 标准化、可复用、热插拔
Agent 能力执行者，具备角色属性 拟人化、可管理、可协作

架构分层

四、核心特性

1. 场景驱动

场景是 Apex 的核心抽象。一个场景代表一个完整的业务上下文：

scene:
  id: customer-service
  name: 智能客服场景
  participants:
    - agent: 客服小美
      role: primary
  capabilities:
    - 问答
    - 工单创建
    - 人工转接
  knowledge:
    - 产品手册
    - 常见问题库
  llm:
    provider: deepseek
    model: deepseek-chat

场景组支持多场景协作，实现复杂业务编排。

2. 技能热插拔

技能是能力的载体，支持完整的生命周期：

发现安装激活调用降级卸载

声明式定义：

apiVersion: skill.ooder.net/v1
kind: Skill
metadata:
  id: skill-qa
  version: 1.0.0
spec:
  capabilities:
    - id: question-answering
      category: intelligence
  dependencies:
    - skill-llm-deepseek

3. Agent 拟人化

每个 Agent 都有角色属性：

属性 说明
名称 身份标识
角色 职责定位
能力 可执行的能力集合
状态 在线/离线/忙碌
指标 负载/成功率/响应时间

协作模式：

  • Super Agent：协调者，负责任务分发
  • Sub Agent：执行者，负责具体任务

4. 多 LLM 集成

内置多 LLM 提供商支持：

Provider 特点
OpenAI 行业标杆
DeepSeek 高性价比
千问 国产优化
Ollama 本地部署

统一 Function Calling 接口，能力动态调用。

5. 知识库增强

支持知识库绑定到场景：

  • 向量嵌入
  • RAG 检索增强
  • 按场景隔离

五、技术创新

A2UI：对话生成动态 UI

用户通过自然语言描述需求，AI 自动生成可交互界面：

六、技术栈升级

维度 升级
语言 Java 8 → Java 21
框架 Spring Boot 2.x → 3.x
存储 SQLite / JSON
API SpringDoc OpenAPI

Java 21 带来的提升：

  • 虚拟线程：高并发处理
  • 模式匹配：代码简洁
  • GraalVM：快速启动

七、差异化优势

维度 传统方案 Apex

Agent 管理 无/简单 拟人化角色系统

  • 企业助手：场景化能力编排
