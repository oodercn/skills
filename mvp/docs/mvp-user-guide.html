MVP 功能手册

从安装到能力激活，一站式新手指南

前言

MVP 是一个场景驱动的 AI Agent 基础设施。本手册将带你从零开始，完成程序的安装、能力的发现、安装与激活全流程。

本手册适用于 MVP 3.0 (Apex) 版本

第一部分：程序安装

1.1 环境要求

项目 要求
Java JDK 21+
内存 128MB ~ 1GB（根据 Profile）
端口 8084（默认）
操作系统 Windows / Linux / macOS

1.2 快速启动

1 下载程序包

从 GitHub Releases 下载最新版本的 mvp-core.jar

# 下载地址
https://github.com/ooder-skills/mvp/releases
2 启动程序 
# Windows
java -jar mvp-core.jar

# Linux/macOS
./mvnw spring-boot:run

# 指定 Profile
java -jar mvp-core.jar --spring.profiles.active=small
3 访问控制台

启动成功后，浏览器访问：

http://localhost:8084
📷 控制台首页截图
截图路径: screenshots/console-home.png

1.3 首次登录

默认管理员账号：

用户名 密码
admin admin123
首次登录后请及时修改默认密码

1.4 Profile 配置

MVP 提供四级部署配置：

Profile 内存 启动时间 包含功能
micro 128MB 5s 核心功能
small 256MB 10s 核心 + 本地 LLM
large 512MB 15s 核心 + LLM + 知识库
enterprise 1GB 30s 全功能

第三部分：能力安装

📖 用户故事（续）

小明在发现结果中找到了「知识问答」能力，现在需要安装它。

3.1 查看能力详情

点击能力卡片，查看详细信息：

  • 能力名称、ID、版本
  • 功能特性描述
  • 支持的角色
  • 依赖项
📷 能力详情截图
截图路径: screenshots/capability-detail.png

3.2 安装向导

点击「安装」按钮，进入安装向导。安装向导包含以下步骤：

步骤 1：预览详情

查看能力的完整信息，确认是否满足需求

步骤 2：选择角色

选择您在场景中的角色，不同角色拥有不同的权限和功能

📷 选择角色截图
截图路径: screenshots/install-role.png

步骤 3：配置参与者

添加场景的参与者：

  • 主导者：拥有激活权限，默认为当前用户
  • 协作者：共同参与场景的其他用户

步骤 4：驱动条件

选择场景的触发条件（如适用）

步骤 5：确认依赖

查看并确认能力所需的依赖项

步骤 6：LLM 配置

配置场景所需的 LLM：

  • 选择 Provider（DeepSeek / OpenAI / 千问 / Ollama）
  • 选择模型
  • 配置参数（Temperature、Max Tokens 等）
  • 配置 Function Calling
  • 绑定知识库（可选）
📷 LLM 配置截图
截图路径: screenshots/install-llm.png

步骤 7：安装进度

等待安装完成，查看安装日志

安装成功后，将显示：

  • 场景信息摘要
  • 为您生成的菜单
  • 通知状态

第六部分：常见问题

Q1：能力发现扫描不到结果？

请检查：

  • 本地 skills 目录是否存在且有内容
  • 网络连接是否正常
  • GitHub/Gitee 访问权限是否配置

Q2：安装失败怎么办？

请查看安装日志，常见原因：

  • 依赖项未满足
  • LLM 配置错误
  • 权限不足

Q3：如何添加协作者？

在安装向导的「配置参与者」步骤，点击「添加协作者」按钮，选择或输入用户 ID

Q4：密钥忘记了怎么办？

可以在「能力详情」页面重新生成密钥
