MVP 用户故事

从能力发现到实际使用的完整旅程

小明 是某科技公司的 IT 管理员。公司最近引入了大量内部知识文档，但员工查找信息效率低下。小明希望找到一个 智能问答系统，帮助员工快速获取所需信息。

目标：在 MVP 平台上发现、安装并激活一个知识问答能力，供团队使用。

1

能力发现

作为 IT 管理员
小明 登录 MVP 控制台，进入能力中心。他选择从 本地目录 和 GitHub 发现能力。点击"开始扫描"后，系统自动搜索可用能力。很快，他发现了"知识问答"能力。
1

进入能力发现页面

点击左侧菜单「能力中心」→「发现能力」

2

选择发现途径

勾选本地技能目录、GitHub 仓库等渠道

3

开始扫描

点击"开始扫描"，系统自动搜索能力

4

查看结果

在发现结果中找到"知识问答"能力

📷
能力发现页面截图
screenshots/01-capability-discovery.png

✓ 故事成果

  • 成功发现 18 个能力
  • 找到目标能力"知识问答"
  • 了解能力的类型和功能描述
2

能力安装

作为 IT 管理员
小明 点击"知识问答"能力的安装按钮。安装向导引导他：选择角色（主导者）、添加协作者（团队成员）、配置 LLM（选择 DeepSeek）。安装顺利完成，系统为他生成了专属菜单。
1

预览详情

查看能力名称、功能特性、支持角色、依赖项

2

选择角色

选择"主导者"角色，拥有激活权限

3

配置参与者

添加团队成员作为协作者

4

驱动条件

选择默认驱动条件

5

确认依赖

确认能力所需的依赖项已满足

6

LLM 配置

选择 DeepSeek 作为 LLM 提供商

7

安装进度

等待安装完成，查看安装日志

📷

✓ 故事成果

  • 成功安装"知识问答"能力
  • 配置了 DeepSeek 作为 LLM
  • 添加了 3 名团队成员作为协作者
  • 系统自动生成了专属菜单
3

能力激活

作为 IT 管理员
小明 进入能力激活页面。他确认参与者信息，获取访问密钥，然后执行网络动作。能力激活成功，团队成员现在可以使用知识问答功能了！
1

确认参与者

确认主导者和协作者信息无误

2

配置条件

选择能力运行的驱动条件

3

获取 KEY

获取访问安全数据的密钥

4

网络动作

执行入网通知，更新能力和待办

5

激活完成

能力已成功激活，可以开始使用

📷
能力激活页面截图
screenshots/04-capability-activation.png

✓ 故事成果

  • 能力激活成功
  • 获取了访问密钥
  • 协作者收到激活通知
  • 能力已添加到"我的能力"列表
4

开始使用

作为团队成员
团队成员小李 收到激活通知后，登录 MVP 控制台。他在"我的能力"中看到了"知识问答"，点击进入，开始提问。系统基于公司知识库，快速给出了准确的答案！
1

查看我的能力

进入「能力中心」→「我的能力」

2

选择能力

点击"知识问答"能力卡片

3

开始对话

输入问题，获取基于知识库的答案

通过这个故事，我们看到了 MVP 平台的完整使用流程：发现能力 → 安装能力 → 激活能力 → 使用能力。整个过程简单直观，让 AI 能力真正像乐高一样自由组合。
