### 3.2 API接口设计

```yaml
# 发现阶段API
DiscoveryAPI:
  - GET  /api/v1/discovery/methods           # 获取发现途径列表
  - POST /api/v1/discovery/scan              # 执行扫描
  - GET  /api/v1/discovery/capabilities/{id} # 获取能力详情
  - POST /api/v1/discovery/preview/{id}      # 预览能力安装配置

# 安装阶段API
InstallAPI:
  - POST /api/v1/installs                    # 创建安装任务
  - GET  /api/v1/installs/{id}               # 获取安装状态
  - POST /api/v1/installs/{id}/execute       # 执行安装
  - POST /api/v1/installs/{id}/rollback      # 回滚安装
  - GET  /api/v1/installs/{id}/logs          # 获取安装日志

# 激活阶段API
ActivationAPI:
  - POST /api/v1/activations                 # 创建激活任务
  - GET  /api/v1/activations/{id}            # 获取激活状态
  - POST /api/v1/activations/{id}/step/{step}# 执行激活步骤
  - POST /api/v1/activations/{id}/complete   # 完成激活
  - POST /api/v1/activations/{id}/skip/{step}# 跳过步骤
```

## 五、讨论要点

### 5.1 需要确认的问题

1. **安装和激活是否需要分开？**
   - 方案A：安装后自动激活（适合简单场景）
   - 方案B：安装后手动激活（适合复杂场景）
   - 建议：根据技能类型自动选择

2. **角色选择时机？**
   - 方案A：安装时选择角色
   - 方案B：激活时选择角色
   - 建议：场景技能在激活时选择，独立技能无需选择

3. **依赖安装策略？**
   - 方案A：自动安装所有依赖
   - 方案B：提示用户确认后安装
   - 建议：必需依赖自动安装，可选依赖提示确认

4. **菜单注册时机？**
   - 方案A：安装完成时注册
   - 方案B：激活完成时注册
   - 建议：激活完成时注册
