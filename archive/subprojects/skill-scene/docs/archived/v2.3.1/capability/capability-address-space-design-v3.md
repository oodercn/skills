# 能力地址空间设计 - 最终版

## 一、总地址空间划分

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        总地址空间划分（参考 Engine v3.0）                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   地址范围           │  区域名称        │  说明                              │
│   ───────────────────┼──────────────────┼───────────────────────────────────│
│   0x0000 - 0x00FF    │  系统区          │  系统核心能力                      │
│   0x0100 - 0x7FFF    │  用户区          │  用户能力                          │
│   0x8000 - 0xFFFF    │  扩展区          │  扩展能力                          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 二、能力区内部结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        能力区内部结构（每个能力区 256 地址）                     │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   每个能力区占用 256 个地址 (0x00 - 0xFF)                                     │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  xx00  │  默认降级服务  │  第一个地址，内置兜底                         │   │
│   │  xx01  │  扩展服务 1    │  驱动/实现 1                                 │   │
│   │  xx02  │  扩展服务 2    │  驱动/实现 2                                 │   │
│   │  xx03  │  扩展服务 3    │  驱动/实现 3                                 │   │
│   │  xx04  │  扩展服务 4    │  驱动/实现 4                                 │   │
│   │  ...   │  ...          │  更多扩展                                    │   │
│   │  xx0F  │  扩展服务 N    │  最后一个扩展                                │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   示例：存储能力区 (0x0200 - 0x02FF)                                         │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  0x0200  │  vfs-local     │  默认降级服务（本地存储）                   │   │
│   │  0x0201  │  vfs-database  │  扩展服务 1（数据库存储）                   │   │
│   │  0x0202  │  vfs-minio     │  扩展服务 2（MinIO存储）                    │   │
│   │  0x0203  │  vfs-oss       │  扩展服务 3（阿里云OSS）                    │   │
│   │  0x0204  │  vfs-s3        │  扩展服务 4（AWS S3）                       │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 四、地址分配规则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        地址分配规则                                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   规则 1: 总地址空间                                                         │
│   ├── 系统区: 0x0000 - 0x00FF (256 地址)                                    │
│   ├── 用户区: 0x0100 - 0x7FFF (约 32K 地址)                                 │
│   └── 扩展区: 0x8000 - 0xFFFF (约 32K 地址)                                 │
│                                                                             │
│   规则 2: 能力区内部                                                         │
│   ├── 每个能力区: 256 地址 (0x00 - 0xFF)                                    │
│   ├── xx00: 默认降级服务（第一个地址）                                        │
│   └── xx01 - xx0F: 扩展服务（驱动/实现）                                     │
│                                                                             │
│   规则 3: 能力区分配                                                         │
│   ├── 每个能力类型分配一个 16 的倍数作为基地址                                │
│   └── 例: org=0x0100, auth=0x0110, vfs=0x0200                               │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 六、总结

### 地址空间划分

| 区域 | 地址范围 | 说明 |
|------|----------|------|
| 系统区 | 0x0000 - 0x00FF | 系统核心能力 |
| 用户区 | 0x0100 - 0x7FFF | 用户能力 |
| 扩展区 | 0x8000 - 0xFFFF | 扩展能力 |

### 能力区内部结构

| 地址 | 说明 |
|------|------|
| xx00 | 默认降级服务（第一个地址） |
| xx01 - xx0F | 扩展服务（驱动/实现） |

### 示例

```
存储能力区 (0x0200 - 0x02FF):
├── 0x0200: vfs-local     # 默认降级（本地存储）
├── 0x0201: vfs-database  # 扩展服务 1
├── 0x0202: vfs-minio     # 扩展服务 2
├── 0x0203: vfs-oss       # 扩展服务 3
└── 0x0204: vfs-s3        # 扩展服务 4
```
