## 一、用户故事概述

### 1.1 用户角色

| 角色 | 说明 | 主要场景 |
|------|------|---------|
| **系统管理员** | 负责系统配置和维护 | 安装技能、配置驱动、监控系统 |
| **业务用户** | 使用场景完成业务目标 | 激活场景、配置参数、查看结果 |
| **开发者** | 开发技能包 | 编写技能、定义能力、发布技能 |

### 1.2 闭环原则

每个用户故事必须形成完整闭环：

```
发现 → 激活 → 配置 → 使用 → 反馈 → 优化
```

## 二、能力管理用户故事

### 2.1 发现能力

**用户故事**: 作为业务用户，我希望发现可用的能力，以便了解系统能提供什么服务。

**闭环流程**:
```
┌─────────────────────────────────────────────────────────────────────────────────┐
│  能力发现闭环                                                                    │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  1. 用户进入能力发现页面                                                        │
│  2. 系统展示已注册的能力列表                                                    │
│  3. 用户按分类/标签筛选能力                                                     │
│  4. 用户查看能力详情（说明、参数、示例）                                         │
│  5. 用户决定是否激活该能力                                                      │
│                                                                                 │
│  菜单入口: 能力管理 → 能力发现                                                  │
│  页面: capability-discovery.html                                                │
│  API: GET /api/v1/capabilities                                                  │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**验收标准**:
- [ ] 能力列表正确显示
- [ ] 分类筛选功能正常
- [ ] 能力详情完整展示
- [ ] 能力说明书文档存在

### 2.2 激活能力

**用户故事**: 作为业务用户，我希望激活需要的能力，以便在场景中使用它们。

**闭环流程**:
```
┌─────────────────────────────────────────────────────────────────────────────────┐
│  能力激活闭环                                                                    │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  1. 用户在能力发现页选择能力                                                    │
│  2. 用户点击激活按钮                                                            │
│  3. 系统检查能力依赖                                                            │
│  4. 用户配置必要参数                                                            │
│  5. 系统激活能力并更新状态                                                      │
│  6. 用户在能力列表看到已激活的能力                                              │
│                                                                                 │
│  菜单入口: 能力管理 → 能力激活                                                  │
│  页面: capability-activation.html                                               │
│  API: POST /api/v1/capabilities/{id}/activate                                   │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

**验收标准**:
- [ ] 激活流程完整
- [ ] 依赖检查正确
- [ ] 参数配置保存
- [ ] 状态更新及时

### 2.3 我的能力

**用户故事**: 作为业务用户，我希望查看已激活的能力，以便管理我的能力集合。

**闭环流程**:
```
┌─────────────────────────────────────────────────────────────────────────────────┐
│  我的能力闭环                                                                    │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  1. 用户进入我的能力页面                                                        │
│  2. 系统展示用户已激活的能力                                                    │
│  3. 用户可以停用不需要的能力                                                    │
│  4. 用户可以修改能力配置                                                        │
│  5. 用户可以查看能力使用统计                                                    │
│                                                                                 │
│  菜单入口: 能力管理 → 我的能力                                                  │
│  页面: my-capabilities.html                                                     │
│  API: GET /api/v1/capabilities?status=ACTIVE                                    │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```
