## 摘要

OoderAgent 是一个革命性的企业级 AI Agent 平台，基于**技能架构（Skills Architecture）**设计理念，让企业能够**零部署、零安装**即可构建自主私有的 AI 能力库。平台内置 **137+ 种标准化技能**，涵盖 LLM 集成、业务流程、知识管理、通讯协作等全场景，开箱即用，支持 P2P 能力分享，真正实现 AI 能力的民主化。

## 五、场景分类详解：三种形态与文件结构

### 5.1 OoderAgent 技能的三种形态

<div class="svg-diagram">
    <img src="diagram_scene_structure.svg" alt="场景分类与文件结构详解" />
</div>

### 5.2 类型一：技能场景（Skill Scenes）

**路径**: `skills/scenes/{scene-name}/`  
**数量**: 16 个预置场景  
**特点**: 可独立运行，依赖 `_business` / `_system` 层  
**用途**: 开箱即用的业务解决方案

**典型文件结构（以 daily-report 为例）**:
```
skills/scenes/daily-report/
├── skill.yaml              # 元数据 + 依赖声明
├── README.md               # 使用说明文档
├── src/main/java/
│   └── net.ooder.scene...   # Java源码（场景逻辑）
├── pom.xml                 # Maven构建配置
└── resources/              # 配置文件、模板等
```

**skill.yaml 示例**:
```yaml
metadata:
  id: daily-report
  name: 日报自动生成
  version: "3.0.1"
spec:
  dependencies:
    - id: skill-llm-config      # 依赖LLM能力
      version: ">=3.0.1"
      required: true
    - id: skill-notification    # 依赖通知能力
      version: ">=3.0.1"
      required: false
```

### 5.3 类型二：独立场景（Independent）

**路径**: `skills/capabilities/` 或 `skills/_drivers/`  
**数量**: 61 个可独立部署模块  
**特点**: 自包含完整 Spring Boot 应用，可单独启动  
**用途**: 外部系统集成点（IM/支付/媒体等）

**典型文件结构（以 skill-im-dingding 为例）**:
```
skills/_drivers/im/skill-im-dingding/
├── skill.yaml              # 完整 spec 定义
├── README.md               # API文档 + 使用示例
├── src/main/java/
│   ├── controller/          # REST API 端点
│   ├── service/             # 业务逻辑实现
│   └── config/              # 配置类
├── pom.xml                 # 含 parent 依赖
└── resources/
    └── application.yml     # 端口、数据库等配置
```

**关键差异**: 独立型 skill 包含完整的 `controller` 和 `application.yml`，可单独监听端口暴露 REST API。

### 5.4 类型三：内部能力（Internal Capabilities）

**路径**: `skills/_base/` + `skills/_system/` + `skills/_business/`  
**数量**: 47 个基础设施模块  
**特点**: 无启动类，纯接口+模型定义  
**用途**: 被其他 skill 依赖消费，不直接暴露 API

**典型文件结构（以 ooder-spi-core 为例）**:
```
skills/_base/ooder-spi-core/
├── skill.yaml              # SPI 接口声明
├── README.md               # 接口设计文档
├── src/main/java/
│   └── net.ooder.spi/
│       ├── im/              # IM 接口定义
│       │   ├── ImDeliveryDriver.java
│       │   └── model/
│       ├── rag/             # RAG 接口定义
│       │   ├── RagEnhanceDriver.java
│       │   └── model/
│       └── workflow/        # 工作流接口定义
│           └── WorkflowDriver.java
├── pom.xml                 # 仅接口依赖
└── META-INF/services/      # SPI 注册文件
    └── net.ooder.spi.im.ImDeliveryDriver
```

### 5.5 三种类型对比总结

| 维度 | 场景型 (Scenes) | 独立型 (Drivers/Caps) | 内部型 (Base/System) |
|------|-------------------|---------------------|---------------------|
| **skill.yaml** | ✅ | ✅ | ✅ |
| **可独立运行** | ✅ | ✅ | ❌ |
| **暴露 REST API** | 可选 | ✅ | ❌ |
| **被依赖消费** | 否 | 可选 | ✅ |
| **数量** | 16 | 61 | 47 (含_base 4个) |
| **加载方式** | 用户触发 / 嵌入主应用 | 独立端口启动 | ClassLoader 加载 |
| **生命周期** | 按需激活 | 服务发现注册 | 被引用时激活 |

### 5.6 依赖关系链

```
scenes (场景型)
    │  depends on
    ▼
_business (内部型 - 业务逻辑)
    │  depends on
    ▼
_system (内部型 - 系统服务)
    │  depends on
    ▼
_base (内部型 - SPI核心接口)
    │
    ├── _drivers (独立型) → _base [可选 _system]
    └── capabilities (独立型) → _base [可选 _system]
```

## 六、技术原理深度解析

### 3.1 SPI 服务发现机制

OoderAgent 采用 JDK 标准 SPI（Service Provider Interface）机制实现模块化：

```
┌─────────────────────────────────────────────────────────────┐
│                    SPI 服务发现架构                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────────┐         ┌─────────────────────┐      │
│   │  接口定义层      │         │    服务实现层        │      │
│   │  (Interface)    │◄────────│   (Implementation)  │      │
│   └────────┬────────┘         └─────────────────────┘      │
│            │                                                │
│            ▼                                                │
│   ┌─────────────────────────────────────────────────────┐   │
│   │           META-INF/services 注册文件                 │   │
│   │                                                      │   │
│   │  net.ooder.spi.im.ImDeliveryDriver                  │   │
│   │  └── net.ooder.skill.im.dingding.DingtalkImDriver   │   │
│   │                                                      │   │
│   │  net.ooder.scene.skill.LlmProvider                  │   │
│   │  ├── DeepSeekProvider                                │   │
│   │  ├── OpenAiProvider                                  │   │
│   │  └── QianwenProvider                                 │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
│   原理：运行时动态加载，自动发现可用服务实现                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 依赖关系自动处理

OoderAgent 内置智能依赖管理：

```yaml
# skill.yaml 依赖声明示例
dependencies:
  - id: ooder-spi-core
    version: ">=3.0.1"
    required: true
    description: "SPI核心接口"
  
  - id: skill-config
    version: ">=3.0.0"
    required: false
    description: "配置服务（可选）"
```

**自动处理机制**：
1. **依赖解析**：启动时自动解析 skill.yaml 依赖树
2. **版本匹配**：支持语义化版本控制（SemVer）
3. **循环检测**：自动检测并阻止循环依赖
4. **懒加载**：非必需依赖按需加载

### 3.3 零部署技术实现

```
┌─────────────────────────────────────────────────────────────┐
│                    ApexOS 零部署架构                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    │
│   │   技能包     │    │   运行时     │    │   服务发现   │    │
│   │  (Skill)    │───►│  (Runtime)  │───►│ (Discovery) │    │
│   │  skill.yaml │    │  Java 21    │    │  SPI机制    │    │

│   2. 启动 ApexOS（java -jar apexos.jar）                    │
```

```bash
# 下载 ApexOS
wget https://github.com/ooderCN/apexos/releases/latest/apexos.jar

java -jar apexos.jar
```

```bash
# 方式 1：复制技能包到目录

# 方式 2：从技能市场安装
apexos skill install skill-llm-deepseek

apexos skill subscribe p2p://enterprise-a/skill-customer-service
```
