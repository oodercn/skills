## 1. 项目概述

### 1.1 项目背景

SkillFlow是一个基于Skill的流程编排系统，旨在将传统的BPM流程引擎与现代的Skill能力体系相结合，实现：

1. **统一的执行模型**：所有执行内容都是Skill的不同形态
2. **场景驱动的流程设计**：场景与Workflow强绑定，实现业务场景的完整生命周期管理
3. **多级多维度管理**：知识库、角色、菜单等配置支持多级管理约束

### 1.2 项目目标

| 目标 | 描述 |
|------|------|
| 统一执行模型 | 执行体=Skill，所有执行内容都是Skill的不同形态 |
| 三维度分类 | Form（形态）× Category（功能）× Provider（提供者） |
| 场景Workflow绑定 | SCENE形态必须绑定WORKFLOW类别 |
| 递进式配置UI | 四层级配置：基础→核心→高级→专家 |
| 多级管理约束 | System→Workflow→Scene→Activity四级约束 |

### 1.3 术语定义

| 术语 | 定义 |
|------|------|
| Skill | 技能，统一的执行单元，所有执行内容都是Skill |
| Form | 形态维度，SCENE（场景）或STANDALONE（独立） |
| Category | 功能维度，LLM/FORM/SERVICE/WORKFLOW等 |
| Provider | 提供者维度，SYSTEM/DRIVER/BUSINESS/USER |
| Scene | 场景，SCENE形态的Skill，必须绑定Workflow |
| Workflow | 流程编排，WORKFLOW类别的Skill |
| Context | 上下文环境，嵌套节点的独立运行环境 |
