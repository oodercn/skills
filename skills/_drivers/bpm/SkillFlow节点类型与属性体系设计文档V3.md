## 1. 核心概念重新定义

### 1.1 活动的本质定义

```
┌─────────────────────────────────────────────────────────────────────┐
│                        活动的本质                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│      活动 = 谁（执行者） + 做什么（执行内容）                         │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  执行者 [Performer]                                          │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • HUMAN  - 真人（需要UI交互）                               │    │
│  │  • AGENT  - 拟人智能体（龙虾，可自主执行或辅助人）            │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              +                                       │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  执行内容 [Execution]                                        │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • SKILL  - 调用技能能力（系统服务、工具调用）                │    │
│  │  • LLM    - 大模型推理（对话、生成、分析）                    │    │
│  │  • FORM   - 传统UI表单（人工填写）                           │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  注意：A2UI 不是独立的节点类型，而是人工任务使用的动态界面技术        │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 执行者与执行内容的组合矩阵

| 执行者 | 执行内容 | 组合结果 | UI技术 | 说明 |
|--------|---------|---------|--------|------|
| HUMAN | FORM | 人工表单任务 | A2UI/传统表单 | 传统审批、填报 |
| HUMAN | SKILL | 人工触发技能 | A2UI按钮触发 | 用户手动调用服务 |
| HUMAN | LLM | 人工对话任务 | A2UI对话界面 | 用户与LLM交互 |
| AGENT | SKILL | Agent执行技能 | 无UI | 自动化服务调用 |
| AGENT | LLM | Agent推理任务 | 无UI | 自动化智能处理 |
| AGENT | FORM | Agent辅助表单 | A2UI预填充 | Agent生成/填充表单 |

### 1.3 场景与活动块的关系（包含关系）

```
┌─────────────────────────────────────────────────────────────────────┐
│                    活动块 [BLOCK]                                    │
│  ─────────────────────────────────────────────────────────────────  │
│  定义：可复用的活动组合单元                                          │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  普通活动块 [BLOCK_NORMAL]                                   │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 可复用的活动组合                                          │    │
│  │  • 无独立生命周期                                             │    │
│  │  • 无独立角色定义                                             │    │
│  │  • 无独立菜单                                                 │    │
│  │  • 随父流程创建/销毁                                          │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│                              ⊂ (包含关系)                            │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  场景 [SCENE] - 特殊的活动块                                 │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 有独立生命周期（激活/运行/暂停/终止）                      │    │
│  │  • 有独立角色定义（USER、ADMIN等）                            │    │
│  │  • 有独立菜单配置                                             │    │
│  │  • 有独立能力清单                                             │    │
│  │  • 有独立依赖服务                                             │    │
│  │  • 可独立部署和运行                                           │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘

结论：场景 ⊂ 活动块（场景是活动块的子类型，不是并行关系）
```

### 1.4 组网节点的设计目的与约束

```
┌─────────────────────────────────────────────────────────────────────┐
│                    组网层设计目的与约束                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  目标：让流程能够安全、可控地延伸到外部                               │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  第一层：协议限制层 [Protocol Constraint]                    │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 必须通过组网协议层才能扩展到外部                           │    │
│  │  • 不能直接访问外部系统                                      │    │
│  │  • 所有外部交互必须经过协议转换                              │    │
│  │  • 协议类型必须在定义期声明                                   │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              ↓                                       │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  第二层：传递层 [Delivery]                                   │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  传递类型：                                                  │    │
│  │  • 命令传递 [COMMAND] - 要求执行某操作（需要响应）            │    │
│  │  • 通知传递 [NOTIFICATION] - 告知某事件发生（无需响应）       │    │
│  │  • 查询传递 [QUERY] - 请求获取信息（需要响应）                │    │
│  │                                                              │    │
│  │  传递执行者：                                                │    │
│  │  • 人 [HUMAN] - 人工确认后传递                               │    │
│  │  • Agent [AGENT] - 自动传递                                  │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              ↓                                       │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  第三层：协同层 [Collaboration]                              │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 跨组织协作                                                │    │
│  │  • 跨系统对接                                                │    │
│  │  • 跨流程联动                                                │    │
│  │  • 跨租户交互                                                │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              ↓                                       │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  第四层：方向层 [Direction] - 南北向协议                     │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 北向 [NORTH]：向上级系统/组织报告                         │    │
│  │  • 南向 [SOUTH]：向下级系统/组织下发                         │    │
│  │  • 东西向 [EAST/WEST]：平级系统/组织协作                     │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 节点类型体系 V3

### 2.1 简化后的节点类型树

```
节点类型树 V3
│
├── 流程控制节点 [CONTROL]
│   ├── 开始节点 [START]
│   │   └── position: START
│   │
│   ├── 结束节点 [END]
│   │   └── position: END
│   │
│   └── 网关节点 [GATEWAY]
│       ├── 排他网关 [XOR_GATEWAY]  → flowControl: { split: 'XOR', join: 'XOR' }
│       ├── 并行网关 [AND_GATEWAY]  → flowControl: { split: 'AND', join: 'AND' }
│       └── 包容网关 [OR_GATEWAY]   → flowControl: { split: 'OR', join: 'OR' }
│
├── 活动节点 [ACTIVITY]
│   │   // 核心：执行者 × 执行内容 的组合
│   │
│   ├── 人工任务 [HUMAN_TASK]
│   │   ├── performer: { type: 'HUMAN', assignee: '...' }
│   │   └── execution: { type: 'FORM' | 'SKILL' | 'LLM', ... }
│   │
│   └── Agent任务 [AGENT_TASK]
│       ├── performer: { type: 'AGENT', agentId: '...' }
│       └── execution: { type: 'SKILL' | 'LLM' | 'FORM', ... }
│
├── 活动块节点 [BLOCK]
│   │   // 场景 ⊂ 活动块（包含关系）
│   │
│   ├── 普通活动块 [BLOCK_NORMAL]
│   │   └── blockType: 'NORMAL', activities: [...]
│   │
│   └── 场景 [SCENE]  // 特殊的活动块
│       ├── blockType: 'SCENE'
│       ├── lifecycle: 'AUTO' | 'TRIGGER' | 'MANUAL'
│       ├── roles: [...]
│       ├── menus: [...]
│       └── capabilities: [...]
│
├── 子流程节点 [SUBFLOW]
│   └── refProcessId: string, params: {...}
│
├── 外部流程节点 [EXTERNAL]
│   └── 通过组网协议对接（必须经过协议层）
│
└── 组网节点 [NETWORK]
    │   // 提供外部扩展能力，但有协议限制
    │
    ├── 协调器 [COORDINATOR]
    │   └── 多Agent协调，内部组网
    │
    ├── 端点 [ENDPOINT]
    │   └── 外部系统接入点，协议转换
    │
    └── 桥接器 [BRIDGE]
        └── 协议转换，消息路由
```

## 3. 组网协议层约定设计

### 3.1 核心问题：定义怎么去约定？

组网协议层的约定需要在**定义期**完成，包括以下几个维度：

```
┌─────────────────────────────────────────────────────────────────────┐
│                    组网协议约定维度                                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  1. 协议约定 [Protocol Contract]                             │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 协议类型：A2A | HTTP | MQTT | KAFKA | WEBSOCKET          │    │
│  │  • 消息格式：JSON | XML | Protobuf | Avro                   │    │
│  │  • 编码方式：UTF-8 | GBK | ...                              │    │
│  │  • 版本控制：v1.0 | v2.0 | ...                              │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  2. 消息约定 [Message Contract]                              │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 消息类型：COMMAND | NOTIFICATION | QUERY                 │    │
│  │  • 消息Schema：输入/输出结构定义                             │    │
│  │  • 消息头：路由、追踪、认证信息                              │    │
│  │  • 消息体：业务数据                                          │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  3. 安全约定 [Security Contract]                             │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 认证方式：TOKEN | CERTIFICATE | OAUTH2 | mTLS            │    │
│  │  • 加密方式：TLS | AES | ...                                │    │
│  │  • 签名验证：HMAC | RSA | ...                               │    │
│  │  • 访问控制：白名单 | 黑名单 | RBAC                          │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  4. 服务约定 [Service Contract]                              │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 超时时间：默认30秒                                        │    │
│  │  • 重试策略：次数、间隔、退避                                │    │
│  │  • 熔断策略：阈值、恢复时间                                  │    │
│  │  • 限流策略：QPS、并发数                                     │    │
│  │  • 降级策略：默认响应、错误处理                              │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  5. 语义约定 [Semantic Contract]                             │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 业务语义：操作含义、预期结果                              │    │
│  │  • 错误码定义：成功、失败、业务错误                          │    │
│  │  • 状态机定义：消息状态流转                                  │    │
│  │  • 幂等性约定：是否支持重试                                  │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 组网协议定义结构

```javascript
const NetworkProtocolContract = {
    protocolId: 'npc_001',
    name: '外部系统对接协议',
    version: '1.0.0',
    
    protocol: {
        type: 'A2A',
        messageFormat: 'JSON',
        encoding: 'UTF-8',
        version: '1.0'
    },
    
    messages: [
        {
            name: 'SubmitApproval',
            type: 'COMMAND',
            direction: 'NORTH',
            description: '提交审批到上级系统',
            schema: {
                input: {
                    type: 'object',
                    properties: {
                        processId: { type: 'string' },
                        activityId: { type: 'string' },
                        payload: { type: 'object' }
                    }
                },
                output: {
                    type: 'object',
                    properties: {
                        success: { type: 'boolean' },
                        externalId: { type: 'string' },
                        message: { type: 'string' }
                    }
                }
            }
        },
        {
            name: 'ApprovalResult',
            type: 'NOTIFICATION',
            direction: 'SOUTH',
            description: '接收上级系统的审批结果',
            schema: {
                input: {
                    type: 'object',
                    properties: {
                        externalId: { type: 'string' },
                        approved: { type: 'boolean' },
                        comment: { type: 'string' }
                    }
                }
            }
        }
    ],
    
    security: {
        auth: {
            type: 'OAUTH2',
            clientId: '${env.OAUTH_CLIENT_ID}',
            clientSecret: '${env.OAUTH_CLIENT_SECRET}',
            tokenUrl: 'https://auth.external.com/oauth/token'
        },
        encryption: 'TLS',
        signature: 'HMAC-SHA256'
    },
    
    service: {
        timeout: 30000,
        retry: {
            maxAttempts: 3,
            interval: 1000,
            backoff: 'EXPONENTIAL'
        },
        circuitBreaker: {
            threshold: 5,
            recoveryTime: 60000
        },
        rateLimit: {
            qps: 100,
            concurrent: 10
        }
    },
    
    semantic: {
        idempotent: true,
        errorCodes: [
            { code: 'SUCCESS', message: '成功' },
            { code: 'AUTH_FAILED', message: '认证失败' },
            { code: 'TIMEOUT', message: '超时' },
            { code: 'BUSINESS_ERROR', message: '业务错误' }
        ]
    }
};
```

## 4. 定义期概念与流程

### 4.1 定义期的核心概念

```
┌─────────────────────────────────────────────────────────────────────┐
│                    定义期核心概念                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
```

### 5.6 方案对比与推荐

| 方案 | 优点 | 缺点 | 适用场景 | 推荐度 |
|------|------|------|---------|--------|
| **统一面板** | 配置集中，效率高 | 复杂度高，学习成本 | 熟练用户 | ⭐⭐⭐⭐ |
| **分步向导** | 引导清晰，易上手 | 步骤多，效率低 | 新用户 | ⭐⭐⭐ |
| **卡片组合** | 直观可视，灵活 | 实现复杂 | 可视化需求 | ⭐⭐⭐⭐ |
| **混合模式** | 兼顾效率与易用 | 实现复杂度高 | 综合场景 | ⭐⭐⭐⭐⭐ |

**最终推荐**：混合模式
- 基础属性：统一面板
- 活动定义：卡片组合（执行者 × 执行内容）
- 高级配置：可折叠面板
- 组网配置：专用配置面板

## 7. 迁移与兼容策略

### 7.1 旧节点类型到新节点类型映射

| 旧类型 | 新类型 | 执行者 | 执行内容 | 说明 |
|--------|--------|--------|---------|------|
| TASK | ACTIVITY | HUMAN | FORM | 人工表单任务 |
| SERVICE | ACTIVITY | AGENT | SKILL | Agent执行技能 |
| SCRIPT | ACTIVITY | AGENT | SKILL | Agent执行脚本 |
| LLM_TASK | ACTIVITY | AGENT | LLM | Agent推理任务 |
| AGENT_TASK | ACTIVITY | AGENT | SKILL/LLM | Agent任务 |
| A2UI | ACTIVITY | HUMAN | FORM | 人工表单（动态UI） |
| SCENE | BLOCK(SCENE) | - | - | 场景（活动块子类型） |
| COORDINATOR | NETWORK | - | - | 组网协调器 |

### 7.2 属性迁移映射

```
旧属性路径                         新属性路径
─────────────────────────────────────────────────────
activityType                 →    (根据映射确定 performer.type + content.type)
activityCategory             →    (废弃)
positionCoord                →    flowDefinition.positionCoord
agentConfig                  →    execution.content.llm 或 execution.content.skill
sceneConfig                  →    flowDefinition.blockConfig (blockType=SCENE)
networkConfig                →    networking.*
```
