## 数据流与校验流程

### 读取流程 (Backend -> Frontend)

```
1. API 获取原始数据
   ↓
2. BpmJsonSchema.Transformer.fromBackend() 转换
   ↓
3. BpmJsonSchema.Validator.validateProcessDef() 校验
   ↓
4. Store 存储到状态
   ↓
5. Canvas 渲染到画布
   ↓
6. Panel 填充到表单
```

### 保存流程 (Frontend -> Backend)

```
1. Panel 收集表单数据
   ↓
2. Store 更新状态
   ↓
3. BpmJsonSchema.Validator.validateProcessDef() 校验
   ↓
4. BpmJsonSchema.Transformer.toBackend() 转换
   ↓
5. API 发送到后端
   ↓
6. 后端存储到数据库
```

### 校验时机

1. **输入时校验**: 表单字段失焦时进行格式校验
2. **保存前校验**: 点击保存按钮时进行完整校验
3. **后端校验**: 后端接收数据后再次校验

### 3. 集成校验器

在 Store.js 中集成校验：

```javascript
class Store {
    saveProcess() {
        const process = this.getProcess();
        
        // 校验数据
        const errors = BpmJsonSchema.Validator.validateProcessDef(process.toJSON());
        if (BpmJsonSchema.Validator.hasErrors(errors)) {
            const errorMsg = BpmJsonSchema.Validator.formatErrors(errors);
            throw new Error(`校验失败:\n${errorMsg}`);
        }
        
        // 转换并发送
        const backendData = BpmJsonSchema.Transformer.toBackend(process.toJSON());
        return this.api.saveProcess(backendData);
    }
}
```
