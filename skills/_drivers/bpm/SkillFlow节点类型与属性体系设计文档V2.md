## 1. 核心概念重新定义

### 1.1 活动的本质定义

```
┌─────────────────────────────────────────────────────────────────────┐
│                        活动的本质                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│      活动 = 谁（执行者） + 做什么（执行内容）                         │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  执行者 [Performer]                                          │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • HUMAN  - 真人（需要UI交互）                               │    │
│  │  • AGENT  - 拟人智能体（龙虾，可自主执行或辅助人）            │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              +                                       │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  执行内容 [Execution]                                        │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • SKILL  - 调用技能能力（系统服务、工具调用）                │    │
│  │  • LLM    - 大模型推理（对话、生成、分析）                    │    │
│  │  • FORM   - 传统UI表单（人工填写）                           │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 1.2 执行者与执行内容的组合

| 执行者 | 执行内容 | 组合结果 | 说明 |
|--------|---------|---------|------|
| HUMAN | FORM | 人工表单任务 | 传统审批、填报 |
| HUMAN | SKILL | 人工触发技能 | 用户手动调用服务 |
| HUMAN | LLM | 人工对话任务 | 用户与LLM交互 |
| AGENT | SKILL | Agent执行技能 | 自动化服务调用 |
| AGENT | LLM | Agent推理任务 | 自动化智能处理 |
| AGENT | FORM | Agent辅助表单 | Agent生成/填充表单 |

### 1.3 场景与活动块的关系

```
┌─────────────────────────────────────────────────────────────────────┐
│                    活动块 [BLOCK]                                    │
│  ─────────────────────────────────────────────────────────────────  │
│  定义：可复用的活动组合单元                                          │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  普通活动块                                                  │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 可复用的活动组合                                          │    │
│  │  • 无独立生命周期                                             │    │
│  │  • 无独立角色定义                                             │    │
│  │  • 无独立菜单                                                 │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  场景 [SCENE] - 特殊的活动块                                 │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 有独立生命周期（激活/运行/暂停/终止）                      │    │
│  │  • 有独立角色定义（USER、ADMIN等）                            │    │
│  │  • 有独立菜单配置                                             │    │
│  │  • 有独立能力清单                                             │    │
│  │  • 有独立依赖服务                                             │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘

结论：场景是活动块的子类型，不是并行关系
```

### 1.4 组网节点的设计目的

```
┌─────────────────────────────────────────────────────────────────────┐
│                    组网层设计目的                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  目标：让流程能够安全、可控地延伸到外部                               │
│                                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  限制层                                                      │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 必须通过组网协议层才能扩展到外部                           │    │
│  │  • 不能直接访问外部系统                                      │    │
│  │  • 所有外部交互必须经过协议转换                              │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              ↓                                       │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  传递层                                                      │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 命令传递（Command）- 要求执行某操作                        │    │
│  │  • 通知传递（Notification）- 告知某事件发生                   │    │
│  │  • 查询传递（Query）- 请求获取信息                            │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              ↓                                       │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  协同层                                                      │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 跨组织协作                                                │    │
│  │  • 跨系统对接                                                │    │
│  │  • 跨流程联动                                                │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                              ↓                                       │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  方向层（南北向协议）                                        │    │
│  │  ─────────────────────────────────────────────────────────── │    │
│  │  • 北向：向上级系统/组织报告                                  │    │
│  │  • 南向：向下级系统/组织下发                                  │    │
│  │  • 东西向：平级系统/组织协作                                  │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 2. 节点类型体系 V2

### 2.1 简化后的节点类型树

```
节点类型树 V2
├── 流程控制节点 [CONTROL]
│   ├── 开始节点 [START]
│   │   └── position: START
│   │
│   ├── 结束节点 [END]
│   │   └── position: END
│   │
│   └── 网关节点 [GATEWAY]
│       ├── 排他网关 [XOR_GATEWAY]
│       ├── 并行网关 [AND_GATEWAY]
│       └── 包容网关 [OR_GATEWAY]
│
├── 活动节点 [ACTIVITY]
│   │
│   │   // 执行者 × 执行内容 的组合
│   │
│   ├── 人工任务 [HUMAN_TASK]
│   │   ├── performer: HUMAN
│   │   └── execution: FORM | SKILL | LLM
│   │
│   └── Agent任务 [AGENT_TASK]
│       ├── performer: AGENT
│       └── execution: SKILL | LLM | FORM
│
├── 活动块节点 [BLOCK]
│   ├── 普通活动块 [BLOCK]
│   │   └── activities: [...]
│   │
│   └── 场景 [SCENE]  // 特殊的活动块
│       ├── lifecycle: AUTO | TRIGGER | MANUAL
│       ├── roles: [...]
│       ├── menus: [...]
│       └── capabilities: [...]
│
├── 子流程节点 [SUBFLOW]
│   └── refProcessId: string
│
├── 外部流程节点 [EXTERNAL]
│   └── 通过组网协议对接
│
└── 组网节点 [NETWORK]
    ├── 协调器 [COORDINATOR]
    │   └── 多Agent协调
    │
    ├── 端点 [ENDPOINT]
    │   └── 外部系统接入点
    │
    └── 桥接器 [BRIDGE]
        └── 协议转换
```

```javascript
    // 外部流程
    EXTERNAL: 'EXTERNAL',
    
    // 组网节点
    NETWORK: 'NETWORK'
};

// 执行者类型
const PerformerType = {
    HUMAN: 'HUMAN',   // 真人
    AGENT: 'AGENT'    // 拟人智能体

// 执行内容类型
const ExecutionType = {
    FORM: 'FORM',     // 传统UI表单

const BlockType = {
    NORMAL: 'NORMAL',   // 普通活动块

};
```

### 4.2 组网协议定义

```javascript
// 组网协议配置
const NetworkProtocol = {
    // 协议类型
    type: 'A2A' | 'HTTP' | 'MQTT' | 'KAFKA' | 'WEBSOCKET',
    
    // 消息类型
    messageTypes: {
        COMMAND: 'COMMAND',           // 命令：要求执行某操作
        NOTIFICATION: 'NOTIFICATION', // 通知：告知某事件发生
        QUERY: 'QUERY'               // 查询：请求获取信息
    },
    
    // 方向
    direction: {
        NORTH: 'NORTH',    // 北向：向上级系统报告
        SOUTH: 'SOUTH',    // 南向：向下级系统下发
        EAST: 'EAST',      // 东西向：平级系统协作
        WEST: 'WEST'
    },
    
    // 认证方式
    auth: {
        type: 'TOKEN' | 'CERTIFICATE' | 'OAUTH2',
        config: {}
    }
};
```

### 5.4 方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| **统一面板** | 配置集中，效率高 | 复杂度高，学习成本 | 熟练用户 |
| **分步向导** | 引导清晰，易上手 | 步骤多，效率低 | 新用户 |
| **卡片组合** | 直观可视，灵活 | 实现复杂 | 可视化需求 |

**推荐**：统一面板 + 卡片组合混合模式
- 基础配置：统一面板
- 执行者×执行内容：卡片组合
- 高级配置：可折叠面板

## 6. 迁移与兼容策略

### 6.1 旧节点类型到新节点类型映射

| 旧类型 | 新类型 | 执行者 | 执行内容 |
|--------|--------|--------|---------|
| TASK | ACTIVITY | HUMAN | FORM |
| SERVICE | ACTIVITY | AGENT | SKILL |
| SCRIPT | ACTIVITY | AGENT | SKILL |
| LLM_TASK | ACTIVITY | AGENT | LLM |
| AGENT_TASK | ACTIVITY | AGENT | SKILL/LLM |
| A2UI | ACTIVITY | HUMAN | FORM (动态) |
| SCENE | BLOCK (SCENE) | - | - |
| COORDINATOR | NETWORK | - | - |

### 6.2 属性迁移映射

```
旧属性路径                         新属性路径
─────────────────────────────────────────────────────
activityType                 →    (根据映射确定 performer + execution)
activityCategory             →    (废弃)
positionCoord                →    flowDefinition.positionCoord
agentConfig                  →    execution.content.llm 或 execution.content.skill
sceneConfig                  →    flowDefinition.blockConfig (blockType=SCENE)
```
